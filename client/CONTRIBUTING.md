## 🎯 كيف يمكنني المساهمة؟

هناك عدة طرق للمساهمة:

### 1. الإبلاغ عن الأخطاء 🐛
إذا وجدت خطأ، يرجى فتح Issue مع:
- وصف واضح للمشكلة
- خطوات إعادة إنتاج المشكلة
- السلوك المتوقع مقابل السلوك الفعلي
- لقطات شاشة (إن أمكن)
- معلومات البيئة (المتصفح، نظام التشغيل)

### 2. اقتراح ميزات 💡
لديك فكرة رائعة؟ افتح Issue مع:
- وصف الميزة المقترحة
- لماذا ستكون هذه الميزة مفيدة
- أمثلة على كيفية استخدامها
- mockups أو تصاميم (اختياري)

### 3. تحسين التوثيق 📚
- تصحيح أخطاء إملائية أو نحوية
- إضافة أمثلة أو شروحات
- ترجمة الوثائق
- تحسين README

### 4. كتابة الكود 💻
- إصلاح الأخطاء
- إضافة ميزات جديدة
- تحسين الأداء
- كتابة اختبارات

## 🐛 الإبلاغ عن الأخطاء

### قبل فتح Issue

1. **ابحث أولاً**: تحقق من Issues الموجودة لتجنب التكرار
2. **تأكد من التحديث**: استخدم آخر إصدار من المشروع
3. **اختبر في بيئة نظيفة**: تأكد أن المشكلة ليست بسبب إعداداتك الخاصة

### نموذج الإبلاغ

```markdown
## وصف المشكلة
وصف واضح ومختصر للمشكلة.

## خطوات إعادة الإنتاج
1. اذهب إلى '...'
2. اضغط على '...'
3. انتقل إلى '...'
4. لاحظ الخطأ

## السلوك المتوقع
ما الذي كنت تتوقع حدوثه؟

## السلوك الفعلي
ما الذي حدث بالفعل؟

## لقطات شاشة
إن أمكن، أضف لقطات شاشة توضح المشكلة.

## البيئة
- المتصفح: [مثال: Chrome 120]
- نظام التشغيل: [مثال: Windows 11]
- الإصدار: [مثال: 1.0.0]

## معلومات إضافية
أي سياق إضافي حول المشكلة.
```

## 🔄 إرسال Pull Request

### قبل البدء

1. **افتح Issue أولاً**: ناقش التغيير المقترح
2. **انتظر الموافقة**: تأكد أن الفريق يوافق على الفكرة
3. **Fork المشروع**: أنشئ نسخة من المشروع

### خطوات العمل

1. **Clone المشروع**
```bash
git clone https://github.com/yourusername/music-platform.git
cd music-platform
```

2. **أنشئ فرع جديد**
```bash
git checkout -b feature/your-feature-name
# أو
git checkout -b fix/your-bug-fix
```

3. **قم بالتغييرات**
- اكتب كود نظيف وموثق
- اتبع معايير الكود
- أضف اختبارات إن أمكن

4. **اختبر التغييرات**
```bash
npm run test
npm run build
```

5. **Commit التغييرات**
```bash
git add .
git commit -m "نوع: وصف مختصر للتغيير"
```

أمثلة على رسائل Commit:
```
feat: إضافة وضع الحفلات
fix: إصلاح مشكلة تشغيل الأغاني
docs: تحديث README
style: تنسيق الكود
refactor: إعادة هيكلة PlayerContext
test: إضافة اختبارات للمشغل
perf: تحسين أداء التحميل
```

6. **Push للـ Fork**
```bash
git push origin feature/your-feature-name
```

7. **افتح Pull Request**
- اذهب إلى صفحة المشروع على GitHub
- اضغط "New Pull Request"
- املأ النموذج بالتفاصيل

### نموذج Pull Request

```markdown
## الوصف
وصف واضح للتغييرات التي قمت بها.

## نوع التغيير
- [ ] إصلاح خطأ (bug fix)
- [ ] ميزة جديدة (new feature)
- [ ] تغيير breaking (breaking change)
- [ ] تحديث توثيق (documentation)

## الاختبار
كيف تم اختبار هذه التغييرات؟

## Checklist
- [ ] الكود يتبع معايير المشروع
- [ ] قمت بمراجعة الكود بنفسي
- [ ] أضفت تعليقات في المناطق الصعبة
- [ ] قمت بتحديث التوثيق
- [ ] لا توجد تحذيرات جديدة
- [ ] أضفت اختبارات للتغييرات
- [ ] جميع الاختبارات تمر بنجاح

## لقطات شاشة (للتغييرات المرئية)
أضف لقطات شاشة قبل وبعد التغيير.

## معلومات إضافية
أي معلومات إضافية مفيدة.
```
